# 📚 目录
# 1️⃣ 大语言模型 (LLM)
## 是什么？
大语言模型就像一个读过全世界所有书的超级学霸。它通过阅读海量文本（互联网、书籍、论文等）学会了理解和生成人类语言。
## 代表模型
## 核心能力
- 文本生成：写文章、故事、代码
- 问答对话：回答各种问题
- 翻译总结：多语言转换、提炼要点
- 推理分析：逻辑思考、问题解决
# 2️⃣ AI Agent (智能体)
## 是什么？
如果说大语言模型是大脑，那么AI Agent就是能动手做事的完整机器人。它不仅能思考，还能调用工具、执行任务、与环境交互。
## Agent vs 大模型
# 3️⃣ 提示词 (Prompt)
## 是什么？
提示词就是你给AI的指令或问题，告诉它你想要什么。就像对服务员点餐——你说得越清楚，上来的菜越符合预期。
## 提示词技巧
❌ 差的提示词：
[帮我写点什么]
✅ 好的提示词：
[请帮我写一封求职邮件，应聘产品经理职位，强调我有3年互联网经验，语气专业但亲切，200字左右]
# 4️⃣ 上下文 (Context)
## 是什么？
上下文就是AI能记住的当前对话内容。就像你和朋友的聊天，需要记住之前说了什么才能继续聊下去。
## 上下文窗口
每个模型能记住的内容量有限，这个限制叫上下文窗口，通常用token来衡量（约1个中文词=1-2个token）。
# 5️⃣ MCP (Model Context Protocol)
## 是什么？
MCP是Anthropic提出的开放标准协议，让AI能安全地连接各种外部工具和数据源。简单说就是[AI的USB接口标准]。
## 为什么重要？
- 统一标准：一次开发，到处使用
- 安全可靠：标准化的权限控制
- 生态繁荣：工具开发者更容易接入
# 6️⃣ Function Calling (函数调用)
## 是什么？
Function Calling是AI调用外部工具的机制。当AI需要查天气、算数学、调API时，它会[调用]对应的函数来完成。
## 工作流程
用户提问 → AI判断需要工具 → 调用函数 → 获取结果 → AI整理回答
# 7️⃣ Skills (技能)
## 是什么？
Skills是封装好的能力模块，教AI如何完成特定任务（如生成PPT、操作文档、搜索网页等）。
## OpenClaw中的Skills
# 8️⃣ CLI (命令行界面)
## 是什么？
CLI是通过文字命令操作电脑的方式（Command Line Interface）。没有图形界面，只有一行行的指令。
## 常用命令示例
`$ ls`
列出当前目录的文件
`$ cd Documents`
进入Documents文件夹
`$ openclaw gateway status`
查看OpenClaw运行状态
# 9️⃣ 工作流 (Workflow)
## 是什么？
工作流是按顺序执行的自动化任务链条。把多个步骤串起来，让AI自动完成复杂流程。
## 示例：写一篇研究报告的工作流
1️⃣ 搜索资料 → 2️⃣ 整理要点 → 3️⃣ 撰写正文 → 4️⃣ 生成PPT → 5️⃣ 发送邮件
# 🔟 RAG (检索增强生成)
## 是什么？
RAG是让AI先查资料再回答的技术。就像开卷考试——AI不是死记硬背，而是实时查找相关信息来作答。
## RAG vs 微调
# 1️⃣1️⃣ 微调 (Fine-tuning)
## 是什么？
微调是在预训练模型基础上继续训练，让模型学会特定领域的知识或风格。就像大学生毕业后接受职业培训。
## 什么时候用微调？
- 需要模型改变[说话风格]（如模拟某个人物）
- 有大量特定领域数据
- 需要模型[内化]知识而非查找
# 1️⃣2️⃣ OpenClaw
## 是什么？
OpenClaw是一个AI自动化平台，让你能用自然语言指挥AI完成各种任务——写文档、查资料、管理文件、自动化工作流等。
## 核心特点
- 自然语言交互：像聊天一样给AI下指令
- Skill系统：可扩展的能力模块
- 多平台集成：飞书、微信、Telegram等
- 私有化部署：数据安全可控
# 1️⃣3️⃣ Claude Code
## 是什么？
Claude Code是Anthropic推出的AI编程助手，可以帮你写代码、改bug、重构项目、执行命令等。
## 和OpenClaw的关系
Claude Code是一个特定场景的工具（编程），而OpenClaw更像是一个通用平台（各种任务）。OpenClaw可以接入Claude Code作为其Agent之一。
# 🗺️ 概念关系总图
理解这些概念如何协作：
📊 三层架构
【基础设施层】
大语言模型(LLM) —— 大脑
【能力扩展层】
Skills + Function Calling + MCP —— 手脚和工具
【应用层】
AI Agent / OpenClaw / Claude Code —— 完整机器人
🔄 典型工作流程
用户输入 → 提示词工程 → 上下文管理 → RAG检索 → LLM推理 → Function Calling → 工具执行 → 结果返回
📋 易混淆概念速查
# 🎉 结语
AI领域的新概念层出不穷，但核心逻辑其实很简单：
- 大模型是大脑（思考）
- Skills/工具是手脚（执行）
- Agent是完整的人（思考+执行）
- 提示词是对话方式
- 上下文是记忆
掌握这些基础概念，你就能更好地理解和使用各种AI工具了。继续探索，享受AI带来的便利吧！

# 1️⃣4️⃣ Routines（例行程序）

## 是什么？
Routines 是把一套多步操作「打包保存」，以后用一个名字就能重复调用。

## 🎮 生活类比
就像游戏里的「宏」——把「先开盾→再放技能→再吃药」录成一个按钮，以后一键触发。

## 实战示例
```bash
# 定义：发布前的检查清单
/routine create precommit --steps "/ultrareview, /test, /lint"

# 使用
/routine precommit
```

# 1️⃣5️⃣ Ultrathink（深度思考）

## 是什么？
让 AI 花费更多 token 用于「思考过程」，而不是急于给答案。Claude Code 2.x 引入。

## 🎓 生活类比
就像考试时「先在草稿纸上把推导过程写清楚，再填答案」。很多难题不写得清楚是出错的根源。

## 使用
- Claude Code：在提示词里加 `ultrathink` 关键字
- 效果：推理步骤更深，复杂架构题准确率 +20%

# 1️⃣6️⃣ Auto Mode（自动模式）

## 是什么？
AI 不再每次询问你，直接执行命令。适合「我信任你」的高重复任务。

## 🚗 生活类比
普通模式 = 学车，每踩油门教练都问「要踩吗？」
Auto Mode = 已经拿驾照，自己开，紧急情况才接管。

## ⚠️ 注意
高风险命令（rm、sudo、推送）仍然需要确认。

# 1️⃣7️⃣ Subagent（子代理）

## 是什么？
让主 AI 启动一个「子 AI」去独立完成子任务，子 AI 跑完把结果交回主 AI。

## 🎮 生活类比
CEO（主 agent）让销售总监（subagent）去谈客户。销售总监谈完回来汇报，CEO 不需要亲自谈。

## 实战价值
- 节省 context（子任务在独立 context 里跑）
- 并行（多个 subagent 同时跑）
- 隔离（subagent 崩溃不影响主 agent）

## 2026 新能力
**Subagent 嵌套**：subagent 可以再 spawn 自己的 subagent，最多 5 层深。
> ⚠️ 嵌套越深烧 token 越狠，2-3 层就够日常用。

# 1️⃣8️⃣ Hooks（钩子）

## 是什么？
在 AI 工具的「生命周期事件」上挂自动脚本。比如「AI 改完文件后自动跑 prettier」。

## 🎣 生活类比
就像智能家居的「自动化」——「门一开就开灯」。Hooks 是 AI 工具的「自动化触发器」。

## 常用事件
- `SessionStart` / `SessionEnd`：session 开始/结束
- `PreToolUse` / `PostToolUse`：工具调用前后
- `UserPromptSubmit`：用户发消息时

# 1️⃣9️⃣ Plugins（插件）

## 是什么？
把 commands + agents + hooks + MCP + UI 打包成一个「扩展包」，跨多个 AI 工具共享。

## 🧩 生活类比
跟浏览器插件、VSCode 扩展一个意思。OpenClaw、Claude Code、Cursor 都支持。

## 代表项目
- **ECC**(Everything Claude Code)：67 agents + 271 skills
- **Superpowers**：7 阶段工作流方法论

# 2️⃣0️⃣ Context Engineering（上下文工程）

## 是什么？
2026 比 prompt engineering 更热的概念。不只是写好提示词，更是**管理好整个上下文**（历史、文件、工具结果）。

## 📚 生活类比
跟律师办案一样——不是临时想起来就说，而是把「案卷、证据、法条、判例」全部整理好，AI 才能给出准的答案。

## 实战技巧
- `/context` 查用量
- `/compact` 手动压缩
- 大任务用 subagent 分担
- 文件内容尽量精简（不要整个文件喂给 AI）
